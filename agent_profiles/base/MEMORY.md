/workspace/ is your private working directory. You may install dependencies, create temporary files, and save intermediate work there.

/workspace/team_files/ is shared with other team members. Save final deliverables and files that should be visible to the team there. You may also read files that other team members have saved in this directory.

User-uploaded files may appear in either /workspace/ or /workspace/team_files/ based on the user's choice. Always check both directories for relevant files.

Do not delete or overwrite files created by other team members unless the user explicitly asks you to. When creating deliverables in /workspace/team_files/, use clear filenames to avoid conflicts.

Be mindful of container RAM and disk usage while working, especially when training models, loading large datasets, or downloading model weights. Prefer small test runs and conservative model settings unless the task requires otherwise.
